Introduction
This document introduces sIBL_GUI, a tool to make easier the process of importing sIBL files in your Target 3D Package. What is sIBL ? It's a short for "Smart IBL" . It's a standard describing all informations and files needeed to provide fast and easy Image Based Lighting Setup. A lot more informations are available here : http://www.smartibl.com.

The Concept
Why doing an external application to load sIBL files in a 3D package and not directly use scripting possibilities of the 3D package ? There are advantages and inconvenients in both methods.
Let's explain how sIBL_GUI works behind : the heart of the application is not sIBL_GUI executable in fact but sIBL_Framework. sIBL_GUI is a frontend for it and call sIBL_Framework each time you need to output a loader script : Basically it uses a Template for your 3D package plus a sIBL file then output the Loader Script that you will have to execute in the Target 3D package and the sIBL will setup alone.
Advantage of doing this way is that all the sIBL parsing, Collection Browsing is handled by sIBL_GUI/Framework, you just just have to deal with a simple Loader Script for each 3D package. For example it took me only a few hours to convert the XSI 7.0 Template in a Maya 2008 one. Rescripting an entire browser with a nice interface would have taken days and I must admit it : I failed to do it for XSI,  that's why I decided to build an external application.
If you want more technicals informations about sIBL_GUI and sIBL_Framework visit this thread : sIBL_Framework And sIBL_GUI Thread.

Installation

Windows Platform :
Installation on Windows is pretty straightforward, just launch "sIBL_GUI_Setup.exe".

Linux Platform :
You will need some QT Libraries in order to run sIBL_GUI under Linux :

  • Debian Distributions ( Ubuntu ) :
    sudo apt-get install libqt4-core
    sudo apt-get install libqt4-gui
  • RPM Distributions ( Fedora ):
    su root
    yum install qt
    yum install qt-x11

You also need a defined "TMPDIR" variable ( a temporary files folder ). Type echo $TMPDIR in a shell, if nothing is printed, that means the variable does not exist, enter export TMPDIR="/tmp" to define it ( You can also put the line in your .bashrc to prevent exporting the variable every time ). Once the variable is defined, uncompress the sIBL_GUI archive into the folder of your choice, CD to the folder, chmod +x both sIBL_Framework and sIBL_GUI then start sIBL_GUI with ./sIBL_GUI from a shell or directly by double clicking sIBL_GUI from your file browser.

Mac Os X Platform :
Download "sIBL_Framework.dmg" and "sIBL_GUI.dmg". Open them and drag "sIBL_Framework" and "sIBL_GUI" into your "Application" folder. Notice that both Applications need to be in the same folder, otherwise you will need to edit sIBL_Framework path in sIBL_GUI preferences.

Usage

An XSI oriented screencast is available here : sIBL_GUI_For_XSI Screencast

  • Collection Wizard

    The first time sIBL_GUI is started, the New Collection Wizard will appear :




    If you choose Ok, you will be prompted to enter a name for your new Collection, otherwise interface will directly start on the Collections Browser Tab.




    After entered the name, a browser tree will popup, choose the root of one of your Collections.




  • Collections Browser Tab



    Once you added your first Collection, interface will start on Collections Browser Tab. This is where you can choose and select the sIBL Set you want to load into your Target 3D Package. You can filter your Global Collection with the three filtering comboBoxes or directly search into it using the Search line.
    The comboBoxes are tri-directionnal : when your choose an item in one of the comboBoxes, the two others items will update to reflect the new Filtered Global Collection content.
    The Search Line will match your entry with the parameters from all your sIBL Sets. You can for example search for the word "Bright" and all sIBL Sets containing "Bright" will show. Comments Field of sIBL Sets are also matched ( making the Comments Field acting as Tags Clouds ).
    Double-click the sIBL Set you want to import and it will be brought to the Import Tab.

    Interactable Widgets Descriptions :

    • Collections ComboBox : Global Collection Filtering by Sub-Collections.
      Tips : You can directly use the mouse scrollwheel over a ComboBox to cycle beetween items without clicking.
    • Authors ComboBox : Global Collection Filtering by Authors.
    • Locations ComboBox : Global Collection Filtering by Locations.
    • Search LineEdit : Searches the input text through the Global Collection ( sIBL Sets Parameters ).
    • Collections List : Current Filtered Collection, double-click an item to send it to Import Tab.
      Tips : Leaving your mouse over a sIBL Set will provide you main informations about it.
    • GPS Map : The GPS Map Shows the location of your sIBL Sets on a World Map. You can Zoom with the ScrollWheel and Pan by Dragging with the Left Mouse Button. Leaving the mouse over a Marker will pop a Preview with some Informations about the sIBL Set. Double Click a Marker will send it to the Import Tab.


  • Import Tab

    Import Tab is where you configure your Loader Script. You can choose your Target 3D Package using one of the Templates, tweak options for the Loader Script, etc... On the left you will find informations and options for the selected sIBL Set, and on the right are options to tweak the Template / Loader Script output.
    Basically, you select the Target 3D Package and the Rendering Engine you want to work with by choosing a Template, you customize your Loader Script options by tweaking the Common / Additional Attributes and you output the Loader Script. Depending your Target 3D Package, it will support Remote Connection allowing you to directly send the Loader Script.

    Interactable Widgets Descriptions :

    • Edit In sIBL Edit PushButton : Sends the sIBL File to sIBL Edit for further editing ( You will need to define a sIBLedit path in the Preferences Tab )
    • Refresh sIBL File PushButton : Reloads the current sIBL Set.
    • Open sIBL Folder PushButton : Opens the folder containing the current sIBL Set.
    • Software ComboBox : Target 3D Package for the Loader Script.
    • Template ComboBox : Template for the Loader Script ( It's often a Rendering Engine ).
    • Open Templates Folder PushButton : Opens the folder containing the Templates.
      Tips : You can define your Custom File Browser in the Preferences Tab. Hardcoded Browsers are : Explorer.exe on Windows, Nautilus, Konqueror, Thunar on Linux and Finder on Mac Os X.
    • Edit Current Template PushButton : Edits the current Template in default Text Editor.
      Tips : You can define your Custom Text Editor in the Preferences Tab. Hardcoded Text Editors are : Notepad.exe on Windows, Gedit, Kwrite, Nedit, Mousepad on Linux and TextEdit on Mac Os X.
    • Common Attributes Page : Common Attributes settings of the Loader Script.
    • Additional Attributes Page : Additional Attributes settings of the Loader Script.
    • Output Loader Script PushButton : Outputs the Loader Script to the "TMP" ( Windows ) or "TMPDIR" ( Linux, MacOsX ) variable path.
    • Open Output Folder Script PushButton : Opens the folder containing the current Loader Script.
    • Send To Software PushButton : Direct connection to Target 3D Package if Remote Connection is available.
    • Ip Address LineEdit : Ip Adress of the computer running the Target 3D Package.
      Tips : 127.0.0.1 is the same as "Localhost" and aims your current computer.
    • Software Port SpinBox : Opened Port of the Target 3D Package.


  • Preferences Tab





    Preferences Tab is where you can define options for sIBL_GUI, Add/Remove Sub Collections, activate the Log Window, etc...

    Interactable Widgets Descriptions :

      Overall Settings :

    • sIBL Edit Path LineEdit : sIBL Edit executable path.
      Tips : You can manually edit the line content, if the path you entered is not valid, the line will reset for the Stored Preferences Value ( This is true for all LineEdit of the Preferences Tab ).
    • sIBL Framework Path LineEdit : Current sIBL_Framework executable path.
    • Custom Text Editor Path LineEdit : Custom Text Editor executable path ( You need to enter the full path for it ).
    • Custom File Browser Path LineEdit : Current File Browser executable path ( You need to enter the full path for it ).
    • Help Path LineEdit : Current Help folder Path.
    • Templates Path LineEdit : Current Templates folder Path.
      Tips : Templates folder is recursively parsed so you can structure it the way you want, just avoid using same names for your Templates.
    • Activate Open GL CheckBox : Activate Open GL Support for the GPS Map.
    • Activate Antialiasing CheckBox : Activate Antialiasing for the GPS Map.
    • Verbose Level ComboBox : Defines sIBL_GUI Verbose Level : Each option comprise the ones over it in the List.
    • Toggles Log Window PushButton : Show The log Window.
      Tips : The Log Window is tearable.

      • Collections Settings :

    • Collections Paths : Current Sub Collections paths.
      Tips : You can manually edit the Table content, if anything you entered is not valid, the line will reset for the Stored Preferences Value. If you entered the same path two times or you have the same sIBL Sets multiple times in different Sub-Collections results can be unpredictable.
    • Edit PushButton : Edit the selected Sub Collection Path.
    • Refresh Collection PushButton : Refresh Global Collection, reparsing all directories, etc...
    • Add PushButton : Adds a new Sub Collection.
    • Remove PushButton : Removes Selected Sub Collections.
      Tips : You can select multiple Sub-Collections at once.

      • Online Update :

    • Check For New Releases PushButton : Launch sIBL_GUI - Updater.
    • Check For New Releases On Startup CheckBox : Launch sIBL_GUI - Updater.
    • Get Latest Online Help PushButton : Launch sIBL_GUI - FTP to Download Online Help.
    • Get Latest Templates PushButton : Launch sIBL_GUI - FTP to Download Templates.


  • Help Tab

    Help Tab is where you can read sIBL_GUI related Manuals.

    Interactable Widgets Descriptions :

    • Available Topics ComboBox : Choose the Manual you want to read.
    • Help Viewer : Displays the chosen Manual.


  • About Tab

    About Tab shows thanks, licensing and legal stuff about sIBL_GUI.

  • Log Window

    Log Window verbose nearly everything happening into sIBL_GUI, it can help finding what's going wrong if sIBL_GUI don't work the way it should.


  • sIBL_GUI - Updater

    sIBL_GUI - Updater allow you to see if your sIBL_GUI Release is up to date. You can check the latest sIBL_GUI version with it's Change Log, and the New / Updated Templates.

    Interactable Widgets Descriptions :

    • Open Repository PushButton : Open the Online Repository in a Web Browser.
    • Get Latest Templates PushButton : Launch sIBL_GUI - FTP to Download Templates.
    • Close PushButton : Close sIBL_GUI - Updater.


  • sIBL_GUI - FTP

    sIBL_GUI - FTP is a small FTP Client that download sIBL_GUI related files.

    Interactable Widgets Descriptions :

    • Start Download PushButton : Starts the related files download.
    • Cancel / Close PushButton : Cancel current Download/ Close sIBL_GUI - FTP.
